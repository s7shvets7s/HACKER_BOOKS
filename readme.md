## About the Repository

**Hacker Books** is my personal collection of technical books, references, standards, lecture materials and other educational resources encountered while studying software development, computer science and information security.

The repository reflects the technologies and subjects I have explored throughout my development journey, from C++ and low-level systems programming to Java backend development, operating systems, algorithms, networks, cryptography and software architecture.

This is not a software product, framework or original book-publishing project. It is an organized technical archive intended to make useful study materials easier to classify, locate and revisit.

> [!NOTE]
> The name **Hacker Books** refers to the original meaning of hacker culture: curiosity, deep technical understanding, experimentation and the desire to understand how computer systems work internally.

---

## 🎯 Purpose

I created this repository to:

* keep technical materials organized in one place;
* preserve references encountered during years of programming study;
* maintain a visible map of the subjects I have explored;
* make it easier to return to important books and standards;
* organize materials by language and technical domain;
* support transitions between different technology stacks;
* preserve both English and Russian editions where available;
* share a categorized collection of potentially useful references.

The repository is primarily a **personal learning archive**, not a curated list claiming that every included book has been read completely or is equally recommended.

## 🧠 Learning Philosophy

My preferred approach is to understand technologies below the surface rather than learning only their public APIs.

For example, studying Java also means exploring:

* how bytecode is executed;
* how classes are loaded;
* how memory is organized inside the JVM;
* how garbage collectors work;
* how reflection accesses program metadata;
* how annotations are processed;
* how dynamic proxies are generated;
* how Spring uses reflection, annotations and proxies internally.

The same principle applies to C++, operating systems, networks and security: practical development is combined with studying implementation details and underlying mechanisms.

## 🔍 How to Use the Library

### Browse Online

The simplest approach is to open one of the topic directories directly through GitHub.

Use the repository search when looking for:

* an author;
* a book title;
* a technology;
* a programming language;
* a specific edition;
* a file format.

### Clone the Entire Repository

```bash
git clone https://github.com/s7shvets7s/HACKER_BOOKS.git
cd HACKER_BOOKS
```

> [!WARNING]
> The repository contains many large binary files. A complete clone may require significant disk space and network traffic.

### Download Only One Category

A sparse clone is preferable when only one section is required:

```bash
git clone --filter=blob:none --sparse https://github.com/s7shvets7s/HACKER_BOOKS.git
cd HACKER_BOOKS
git sparse-checkout set java
```

For another category, replace `java` with its directory name:

```bash
git sparse-checkout set Algorithms
```

Directories containing spaces should be quoted:

```bash
git sparse-checkout set "linux and git"
```
